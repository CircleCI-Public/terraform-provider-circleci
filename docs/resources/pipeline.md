# circleci_pipeline (Resource)

Manages a CircleCI pipeline definition. Pipeline definitions are used by GitHub App (standalone) projects to configure where the CI configuration file is located and which repository to check out.

OAuth-based projects (using the `gh/` or `bb/` slug prefixes) do not need pipeline definitions — they use implicit configuration from `.circleci/config.yml` in the repository.

## Example Usage

```terraform
resource "circleci_pipeline" "example" {
  project_id  = circleci_project.example.id
  name        = "my-pipeline"
  description = "Main CI/CD pipeline"

  config_source_provider         = "github_app"
  config_source_file_path        = ".circleci/config.yml"
  config_source_repo_external_id = "123456789"

  checkout_source_provider         = "github_app"
  checkout_source_repo_external_id = "123456789"
}
```

-> The `repo_external_id` values are the GitHub-internal numeric repository IDs. You can find them via the GitHub API: `GET /repos/{owner}/{repo}` returns the `id` field.

## Schema

### Required

- `checkout_source_provider` (String) The VCS provider for the pipeline's checkout source. Must be `github_app`.
- `checkout_source_repo_external_id` (String) The external (GitHub) repository ID for the checkout source.
- `config_source_file_path` (String) Path to the CircleCI configuration file within the repository (e.g. `.circleci/config.yml`).
- `config_source_provider` (String) The VCS provider for the pipeline's configuration source. Must be `github_app`.
- `config_source_repo_external_id` (String) The external (GitHub) repository ID for the configuration source.
- `description` (String) A human-readable description of the pipeline definition.
- `name` (String) The name of the pipeline definition. Changing this forces recreation of the resource.
- `project_id` (String) The ID of the CircleCI project this pipeline definition belongs to.

### Read-Only

- `checkout_source_repo_full_name` (String) The full name (e.g. `org/repo`) of the checkout source repository, as returned by the API.
- `config_source_repo_full_name` (String) The full name (e.g. `org/repo`) of the config source repository, as returned by the API.
- `created_at` (String) Timestamp of when the pipeline definition was created.
- `id` (String) The unique identifier of the pipeline definition.

## Import

Pipeline definitions can be imported using the format `project_id/pipeline_id`:

```shell
terraform import circleci_pipeline.example "PROJECT_ID/PIPELINE_ID"
```
