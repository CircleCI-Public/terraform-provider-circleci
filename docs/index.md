# CircleCI Provider

Use the CircleCI provider to interact with CircleCI resources. The provider enables Infrastructure as Code (IaC) management of CircleCI projects, contexts, pipelines, webhooks, and environment variables.

You must configure the provider with proper credentials before you can use it.

## Example Usage

```terraform
terraform {
  required_providers {
    circleci = {
      source  = "CircleCI-Public/circleci"
      version = "~> 0.1"
    }
  }
}

provider "circleci" {
  host = "https://circleci.com/api/v2"
  key  = var.circleci_token
}
```

## Authentication

The provider requires a CircleCI personal API token. The token can be provided via:

1. The `key` attribute in the provider configuration
2. The `CIRCLE_TOKEN` environment variable

!> **Warning:** Hard-coded credentials are not recommended and risk secret leakage if committed to version control.

### Using Environment Variables (Recommended)

```terraform
provider "circleci" {}
```

```console
$ export CIRCLE_TOKEN="your-api-token"
$ terraform plan
```
